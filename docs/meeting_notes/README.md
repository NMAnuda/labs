# Meeting Notes

A record of important project meetings, decisions, and action items.

## Purpose

Maintaining meeting notes ensures decisions are documented, action items are tracked, and team members who couldn't attend can stay informed. This is especially important for sprint planning, retrospectives, and architecture discussions.

## Naming Convention

Use the following format for meeting note files:

```
YYYY-MM-DD_[Meeting_Type].md
```

**Examples:**
- `2025-01-15_Sprint_Planning.md`
- `2025-01-22_Architecture_Review.md`
- `2025-02-01_Retrospective.md`

## Meeting Types

| Type                | Cadence      | Purpose |
|---------------------|-------------|---------|
| Sprint Planning     | Per sprint  | Plan upcoming sprint work |
| Daily Standup       | Daily       | Quick sync (notes optional) |
| Sprint Review       | Per sprint  | Demo completed work |
| Retrospective       | Per sprint  | Reflect and improve processes |
| Architecture Review | As needed   | Discuss significant technical decisions |
| Stakeholder Update  | Bi-weekly   | Update stakeholders on progress |

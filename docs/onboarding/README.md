# Onboarding

Documentation and resources to help new team members get up to speed quickly.

## Purpose

This folder contains everything a new developer, designer, or project stakeholder needs to become productive on this project. A well-maintained onboarding guide reduces ramp-up time and ensures consistency across the team.

## Onboarding Checklist

### Day 1 — Access & Environment

- [ ] Gain access to the GitHub repository
- [ ] Gain access to project management tools (e.g., Jira, Linear, Trello)
- [ ] Gain access to communication channels (e.g., Slack, Teams)
- [ ] Clone the repository and follow the [Getting Started](../../README.md#getting-started) guide
- [ ] Set up local development environment
- [ ] Copy `.env.example` to `.env` and configure

### Day 1–2 — Project Context

- [ ] Read the [README](../../README.md) for project overview
- [ ] Review the [BRD](../01_BRD_Template.md) — Business Requirements
- [ ] Review the [PRD](../02_PRD_Template.md) — Product Requirements
- [ ] Review the [FSD](../03_FSD_Template.md) — Functional Specification
- [ ] Review the [SDD](../04_SDD_Template.md) — Software Design
- [ ] Watch any **Video Walkthroughs** linked in the documents

### Day 2–3 — Technical Deep Dive

- [ ] Review the [API Documentation](../05_API_Documentation_Template.md)
- [ ] Review the [Deployment Guide](../06_Deployment_Guide_Template.md)
- [ ] Review the [Test Strategy](../07_Test_Strategy_Template.md)
- [ ] Review [Architecture Decision Records](../adr/README.md)
- [ ] Run the test suite locally
- [ ] Make a small test change and submit a PR using the [PR Template](../../.github/PULL_REQUEST_TEMPLATE.md)

### Day 3–5 — First Contribution

- [ ] Pick up a starter issue (labeled `good-first-issue` or similar)
- [ ] Pair with a team member on first PR
- [ ] Get first PR reviewed and merged
